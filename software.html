Computer Architecture

Introduction to Computer Architecture and the Functionality

• Numerous significant inventions that changed how people use and interact with computers have influenced the development of computers and computing technologies. Here are some significant advancements that have helped the development of computing technologies: Transistors; Integrated circuits; Graphical user interface; The internet; Mobile computing

These are only a few of the numerous advancements that have helped computers and computing technologies advance. We may anticipate future developments in fields like artificial intelligence, virtual and augmented reality, and quantum computing as technology develops, which will continue to change how we use and interact with computers.

• Computer structure components are the physical components of a computer system that work together to process and store data. Here are some of the critical functional issues of computer structure components: Central Processing Unit (CPU); Memory; Storage; Input/Output; Motherboard

These are just a few of the critical functional issues of computer structure components. Ensuring that each component is optimized for these functional issues is important to ensure that the computer system can perform effectively and efficiently.

• The process of displaying data in a form that a computer can comprehend and interpret is known as data representation. The nature of data representation for computers is subject to various ideas. The following are some of the main ideas: Binary representation; Number systems; ASCII and Unicode; Data types; Endianness.

Computer workers should be familiar with the theories and principles of data representation since it can help to ensure that data is appropriately processed and stored by computer systems. Additionally, it can aid in enhancing the functionality and effectiveness of computer systems.

Unit 4

Computing Infrastructure

Computing Infrastructure and Network

• The internet and intranets are two distinct networking techniques that are used in diverse contexts. Their distinctions are as follows: The internet is a global network of connected computers and servers that communicate with one another using the standard Internet Protocol (IP), whereas an intranet is a private network designed exclusively for use within a company or a small group of people.

In conclusion, an intranet is a private network created specifically for use within a company or a certain group of people, whereas the internet is a global network that enables individuals and businesses to access and share information. The primary distinction between the two is that whereas an intranet is a closed network that only authorised users can access, the internet is a public network that is available to everyone.

• A network's logical or physical layout is referred to as its topology. There are numerous network topologies, each with unique benefits and drawbacks. Some of the most typical network topologies are: Bus Topology, Ring Topology, Star Topology, Mesh Topology, and to make these networks work, various technologies and protocols are used, such as: Ethernet, Wi-Fi, TCP/IP and VPN

In summary, there are several network topologies, each with its own advantages and disadvantages. To make these networks work, various technologies and protocols such as Ethernet, Wi-Fi, TCP/IP, and VPN are used.

• The choice of a network architecture is influenced by a number of variables, including the network's size, the number of connected devices, and the desired level of security. It is crucial to take into account any potential network security vulnerabilities when choosing a topology.

A star topology is a good choice for small to medium-sized networks, but it is important to be aware of the cybersecurity threats associated with it. To mitigate these threats, it is essential to implement proper security measures such as firewalls, access controls, and encryption. Additionally, regular network monitoring and patching can help detect and fix vulnerabilities before they can be exploited by attackers.

Unit 5

Software as a Tool

Software as a Tool - Its Functionality and Development Methodologies

• On the basis of its functionality and intended use, the software can be divided into various categories. Here are a few typical software categories: System software, Programming software, Web software and Gaming software.

In conclusion, software can be divided into many groups according to its functionality and use. When choosing software as a tool for problem-solving in the workplace, it's crucial to take the organization's unique goals and requirements into account, as well as the software's features, user experience, compatibility, cost, and support.

• Software can be designed, developed, and tested in collaboration using a variety of software development approaches. Every methodology includes a unique set of procedures and guidelines that can be customised to meet the particular requirements of a project. Here are the four well-known software development techniques along with examples of how they might be used in the software development lifecycle: Waterfall Methodology; Agile Methodology; DevOps Methodology; Scrum Methodology

Software development approaches can be modified to meet the unique requirements of a project. The Scrum technique is an agile framework for software development, the DevOps methodology stresses communication between development and operations teams, the waterfall methodology is a linear and sequential approach, and the agile methodologies are iterative and incremental. Each technique has benefits and can be used to efficiently cooperate on software design, development, and testing.

• To guarantee that software is of high quality and complies with requirements, a variety of software validation and correctness approaches can be used. Popular ones include the following: Unit testing; Integration testing; System testing; Acceptance testing; and Automated testing

The selection of a methodology is based on both the stage of the software development process and the particular requirements of the software. Unit testing, for instance, is better suited for early phases of development, whereas system and acceptability testing are better suited for later phases. Additionally, a mix of approaches may be employed to guarantee that the programme is adequately tested and validated for accuracy.
